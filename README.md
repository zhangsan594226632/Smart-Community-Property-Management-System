## 一、项目整体概述
### 1\.1 项目简介

本项目**智慧社区物业管理系统**是一套面向小区物业管理人员的后台综合管理平台，采用行业主流前后端分离架构开发，完整覆盖社区全生命周期物业业务：资产台账管理、水电抄表、物业 / 车位收费、业主投诉报修工单、数据统计可视化、账号权限管控。
系统解决传统小区物业线下纸质登记、收费核算繁琐、工单处理无追踪、社区资产难统计、业务数据无可视化分析等行业痛点，将楼宇、房屋、业主、车位、费用、工单全部线上数字化管理，页面采用标准企业级后台 UI，操作逻辑贴合物业工作人员日常办公习惯，支持多条件筛选、批量操作、数据图表可视化，大幅提升物业办公效率。

### 1\.2 开发软硬件环境

#### 后端技术栈

1. 核心框架：SpringBoot 2\.7，快速搭建 Web 服务，内置 Tomcat 容器

2. 持久层框架：MyBatis\-Plus，封装通用 CRUD、分页、批量操作，减少手写 SQL

3. 安全体系：登录拦截器、角色权限控制、MD5 不可逆密码加密、表单参数双层校验

4. 工具类封装：分页工具、日期工具、费用计算工具、批量删除工具、弹窗表单校验工具

5. 接口规范：统一 RESTful 接口，标准 JSON 返回格式，全局异常捕获处理

#### 前端技术栈

1. 核心框架：Vue3 \+ Vite，轻量化构建，页面渲染速度更快

2. UI 组件库：Element Plus，提供表格、弹窗、分页、下拉选择、图表、表单全套组件

3. 网络请求：Axios 全局封装请求拦截、响应拦截，统一添加 token 鉴权、错误提示

4. 可视化图表：ECharts，实现报修类型柱状统计图、数据指标卡片统计

5. 通用组件：全局复用搜索栏、分页组件、批量操作按钮、新增编辑弹窗

#### 数据库

MySQL 8\.0，字符集 utf8mb4，支持中文、长文本、特殊字符；引擎 InnoDB，支持事务保证收费、工单数据操作一致性。

#### 开发 \& 运行工具

后端开发：IDEA Ultimate
前端开发：VS Code
数据库可视化：Navicat Premium
浏览器：Chrome、Edge
构建工具：Maven 3\.6、Node\.js 16

### 1\.3 系统角色划分

系统分为两类操作角色，登录页通过下拉选择区分，权限完全隔离：

1. **管理员（admin）**
物业超级管理员，拥有系统全部功能权限：基础信息（楼宇 / 房屋 / 业主 / 车位）管理、抄表登记、物业 / 车位收费、投诉工单、报修工单、数据统计分析、后台用户账号管理，支持新增、编辑、单条 / 批量删除、业务处理操作。

2. **业主（普通住户）**
仅前台访问权限（本截图展示管理员后台），可提交投诉、报修、查看个人费用账单，无后台数据管理、批量删除、资产编辑权限。

### 1\.4 项目整体业务价值

1. 资产数字化：统一管理楼宇、房屋、业主、车位，建立完整社区电子台账；

2. 收费标准化：水电抄表自动核算用量，物业费、车位费线上登记缴费状态；

3. 工单闭环化：业主提交投诉 / 报修，物业后台接单处理，全程记录处理人、时间、状态，工单可追溯；

4. 数据可视化：首页大盘 \+ 报修统计图表，直观展示收费收入、工单总量、处理效率；

5. 操作高效化：批量删除、多条件组合检索、快捷入口，减少物业重复手工登记工作。

### 3\.1 登录模块

页面名称：智慧社区物业管理系统登录页

#### 页面元素

1. 顶部标题：智慧社区物业管理系统；

2. 表单输入项：登录账号输入框、密码输入框（带密码显隐切换图标）、用户类型下拉选择框（管理员 / 业主）；

3. 蓝色「立即登录」主按钮；

4. 底部温馨提示：预设默认账号

    - 管理员账号：admin / 123456

    - 业主账号：mary / 123456

#### 业务逻辑与校验

1. 前端校验：账号、密码、用户类型不能为空，空白点击登录弹出提示；

2. 后端校验：查询数据库匹配账号，MD5 加密比对密码；校验角色身份；

3. 登录鉴权：登录成功生成 token 存入前端缓存，跳转对应端页面；未登录访问后台所有页面自动拦截跳转登录页；

4. 安全机制：密码采用不可逆 MD5 加密存储，数据库不保存明文密码。

### 3\.2 管理员后台首页（系统总览仪表盘）

页面名称：系统首页 \- 今日概览

#### 页面布局

左侧固定导航菜单栏 \+ 顶部用户名右上角下拉 \+ 主体数据看板区域

#### 主体四大功能分区

1. **欢迎标题区**
展示登录用户名、系统名称「智慧社区物业管理系统 — 今日概览」；

2. **社区资产统计卡片**
4 个彩色数据卡片，自动统计数据库实时总量：

- 楼宇总数、房屋总数、业主总数、车位总数；

3. **待办事项模块（业务预警）**
自动筛选待处理工单、未缴费费用，每条待办附带紧急 / 重要 / 普通标签，右侧「去处理 / 去查看」快捷按钮，一键跳转对应业务页面：

- 紧急：待处理投诉工单数量

- 重要：待处理报修工单数量

- 普通：物业费未缴记录、车位费到期提醒

4. **本月数据概览财务卡片**
四块财务指标自动汇总当月收费数据：

- 本月物业收入、本月车位费收入、物业费收缴率、本月报修工单总量；

5. **快捷入口功能区**
6 个业务快捷按钮，直达高频操作页面：车位收费、物业收费、投诉管理、报修管理、楼宇管理、统计分析；

#### 技术实现亮点

1. 页面加载自动请求后端聚合统计接口，一次性返回所有资产、财务、工单统计数据，减少多次接口请求；

2. 待办事项实时动态查询，打开首页自动刷新未处理业务；

3. 左侧菜单权限联动，管理员展示全部菜单，业主仅展示个人服务菜单。

### 3\.3 核心业务处理 \- 物业收费管理

页面名称：物业收费管理

#### 1\. 顶部复合搜索栏

双条件筛选：房间号输入框、缴费状态下拉选择（已缴费 / 未缴费），搭配「搜索 / 重置」按钮，支持多条件联合查询，清空条件一键重置列表。

#### 2\. 批量操作按钮

绿色「新增」单条录入收费记录、红色「批量删除」勾选多条费用记录统一删除。

#### 3\. 收费数据列表字段

序号、房间号、房东（业主）、费用类型（水费 / 电费 / 物业费）、计费起止时间、金额 \(元\)、缴费状态（绿色已缴费 / 橙色未缴费）、操作栏。

#### 4\. 操作栏功能

- 编辑：弹窗修改当前收费记录信息；

- 缴费：仅未缴费记录显示，点击更新缴费状态为「已缴费」；

- 删除：单条删除本条收费数据；

#### 业务流程配套逻辑

1. 抄表模块录入水电用量后，自动生成对应水费、电费收费记录；物业费可手动新增；

2. 缴费状态永久存储，首页仪表盘自动统计当月已缴总额、收缴率；

3. 未缴费记录自动推送至首页待办事项提醒物业催收。

### 3\.4 核心业务处理 \- 投诉信息管理

页面名称：投诉信息管理

#### 1\. 多维度筛选搜索栏

双下拉筛选：投诉类型、处理状态（待处理 / 已处理），支持组合条件查询、重置；

#### 2\. 批量操作：批量删除勾选多条投诉工单；

#### 3\. 列表字段

序号、投诉类型、投诉内容、投诉人、投诉时间、处理状态、处理人、操作；
状态区分：橙色待处理、绿色已处理；

#### 4\. 操作按钮

- 投诉处理：仅待处理工单可见，弹窗录入处理人、处理方案，更新状态为已处理；

- 删除：单条工单删除；

#### 业务闭环流程

1. 业主前台提交投诉工单，存入数据库默认「待处理」；

2. 管理员首页待办自动展示待处理投诉，点击跳转本页面；

3. 物业完成处理后点击「投诉处理」，录入处理人信息，工单归档；

4. 所有历史工单永久留存，支持检索追溯社区居民投诉记录。

### 3\.5 核心业务处理 \- 报修信息管理

页面名称：报修信息管理

#### 1\. 筛选栏：报修类型下拉、处理状态下拉，组合检索；

#### 2\. 批量删除功能；

#### 3\. 列表字段

报修类型（电梯 / 自用部位维修 / 电路 / 共用设施设备）、报修内容、报修人、报修时间、处理状态、处理人、操作；

#### 4\. 操作功能

报修处理（待处理工单）、删除；

#### 业务场景覆盖

覆盖小区全品类维修需求：电梯故障、电路故障、阳台水管、门窗、健身器材等公共 / 私人设施维修；处理完成记录维修师傅姓名，形成完整维修台账。

### 3\.6 基础信息管理 \- 房屋资产台账管理

页面名称：房屋管理

#### 1\. 复合搜索条件：房间号输入框、所属楼宇下拉筛选；

#### 2\. 操作按钮：添加房屋、批量删除；

#### 3\. 列表字段

房间号、所属楼宇、房屋面积㎡、户主（绑定业主账号）、入住状态（已入住）、编辑 / 删除操作；

#### 业务作用

建立小区房屋电子档案，所有收费、投诉、报修数据均关联房屋 ID，实现一户一档数据联动；新增房屋时绑定对应楼宇、业主，为后续收费、工单提供基础数据支撑。

### 3\.7 基础信息管理 \- 抄表管理（水电用量登记）

页面名称：抄表管理，核心弹窗「抄表登记」

#### 弹窗表单字段

门牌号、户主、费用类型下拉（水费 / 电费）、上次度数、本次度数、抄表员、登记时间；

#### 自动计算逻辑

系统后台自动计算用量 = 本次度数 \- 上次度数，生成对应水费 / 电费收费记录，同步至物业收费管理列表；

#### 业务价值

替代线下纸质抄表本，线上登记水电读数，自动生成费用单据，减少人工核算误差。

### 3\.8 数据统计分析 \- 报修统计分析可视化页面

页面名称：报修统计分析

#### 1\. 顶部数据指标卡片（实时统计）

报修总数、已处理工单数量、待处理工单数量、平均处理时长；

#### 2\. 报修类型分布柱状统计图

ECharts 可视化图表，直观展示各类报修占比：电梯维修、自用部位、公共设施、电路维修等分类数量，辅助物业调配维修人力；

#### 3\. 报修统计明细表格

支持按报修类型、处理状态筛选，展示全部报修工单明细，可导出打印做月度运维报表；

#### 技术亮点

后端聚合统计 SQL，一次性统计各类报修数量，前端 ECharts 渲染柱状图，实现数据可视化分析，为物业运维决策提供数据支撑。

### 3\.9 其他配套基础管理模块

1. **楼宇管理**：录入小区楼栋编号、楼栋名称，房屋绑定楼宇外键；

2. **业主管理**：维护住户基础信息（姓名、手机号、生日、性别、登录账号），新增 / 编辑 / 批量删除；

3. **车位管理**：登记车位编号、绑定业主、车位到期时间，首页自动提醒到期车位；

4. **车位收费管理**：登记车位租赁费、缴费状态，统计车位月度收入；

5. **后台用户管理**：维护物业后台管理员账号，分配操作权限。

## 五、项目核心技术亮点

### 亮点 1：标准前后端分离架构，业务完全解耦

后端 SpringBoot 只输出 RESTful JSON 接口，不参与页面渲染；Vue3 前端独立负责页面展示、交互、图表渲染。
优势：

1. 前端 UI 改版、页面调整无需修改后端代码；

2. 后端可单独提供接口给业主小程序、H5 页面复用；

3. 开发分工清晰，后端专注业务逻辑、数据库，前端专注交互体验。

### 亮点 2：全局权限拦截 \+ 双角色隔离，系统数据安全可控

1. 登录拦截器全局生效，未登录用户访问任何后台页面强制跳转登录；

2. 角色区分：管理员完整后台权限，业主仅能访问前台个人服务，无法进入资产、收费、工单管理页面；

3. 接口层二次校验权限，即使前端绕过菜单，后端也会拦截越权操作，杜绝数据泄露、恶意删除。

### 亮点 3：MyBatis\-Plus 简化数据库开发，内置批量操作能力

1. 基础增删改查、分页、条件查询无需手写 SQL，大幅缩短数据库开发周期；

2. 统一封装批量删除接口，房屋、业主、收费、工单均支持勾选多条批量清理，适配物业批量台账整理需求；

3. 动态 SQL 拼接多条件复合查询（房间号 \+ 楼宇、投诉类型 \+ 状态），无需复杂 xml 语句。

### 亮点 4：水电抄表自动核算，收费业务自动化联动

抄表登记弹窗录入两次读数，后端自动计算用水量 / 用电量，同步生成对应水费、电费收费单据，无需物业手工计算金额，降低人工计算失误，实现抄表 \- 收费一体化流程。

### 亮点 5：工单全闭环数字化管理，业务全程可追溯

投诉、报修工单实现完整业务闭环：业主提交→后台待办预警→物业接单处理→归档留存，每条工单记录提交人、时间、处理人、处理状态，支持长期检索，满足物业台账归档、业主纠纷溯源需求。

### 亮点 6：ECharts 数据可视化，物业经营数据直观展示

1. 首页仪表盘聚合社区资产、月度收费、待办工单核心指标卡片；

2. 报修统计页面集成柱状图，分类展示各类维修工单数量，辅助物业调配维修人力；

3. 所有统计数据由后端聚合接口一次性返回，前端无需多次请求数据库，性能更优。

### 亮点 7：复合多条件检索，高效筛选业务数据

所有业务列表（收费、投诉、报修、房屋、抄表）均支持**多条件联合筛选**：文本输入框 \+ 下拉选择框组合查询，支持一键重置清空条件，快速定位目标房屋、工单、收费记录，适配物业日常高频查询需求。

### 亮点 8：统一弹窗表单 \+ 全局表单校验，交互规范统一

1. 新增、编辑、抄表登记全部使用统一弹窗表单，页面布局风格标准化；

2. 前端实时校验必填项、数字范围、日期格式，后端二次参数校验，非法数据无法入库，保证数据库数据规范；

3. 全局统一操作提示：新增成功、缴费成功、处理完成、删除确认弹窗提示，交互反馈清晰。

### 亮点 9：统一费用、工单状态标签视觉区分

页面使用色彩区分业务状态：

- 绿色标签：已缴费 / 已处理；

- 橙色标签：未缴费 / 待处理；
视觉直观，物业工作人员一眼区分待办业务，提升办公效率。

### 亮点 10：首页待办自动预警，减少人工定期核查

系统自动统计未处理投诉、未处理报修、未缴物业费、到期车位，首页集中展示待办事项并提供一键跳转入口，替代物业人工逐条翻阅台账查找待办业务，降低漏处理风险。

## 六、核心业务完整流程说明

### 6\.1 水电抄表→生成收费记录流程

1. 管理员登录后台，左侧菜单打开「抄表管理」；

2. 点击「抄表登记」弹出表单，填写门牌号、户主、费用类型、上次 / 本次读数；

3. 点击保存，后端自动计算用量，同步新增一条水费 / 电费收费记录至「物业收费管理」页面；

4. 首页仪表盘自动更新本月水费电费收入、未缴费待办提醒。

### 6\.2 投诉工单处理业务流程

1. 业主前台提交投诉，投诉数据存入数据库，处理状态默认为「待处理」；

2. 管理员登录首页，待办区域展示投诉待办数量，点击「去处理」跳转投诉管理页面；

3. 通过投诉类型、状态筛选待处理工单，点击「投诉处理」弹窗录入处理人；

4. 保存后工单状态更新为「已处理」，永久留存工单记录，可随时检索历史投诉。

### 6\.3 房屋资产录入流程

1. 管理员先进入「楼宇管理」新增小区楼栋信息；

2. 进入「业主管理」录入住户基础账号信息；

3. 打开「房屋管理」点击添加房屋，填写房间号、面积，选择所属楼宇、绑定户主业主；

4. 房屋录入完成后，抄表、收费、工单均可关联对应房屋数据，实现一户一档管理。

### 6\.4 报修数据统计分析流程

1. 业主提交各类维修工单，系统持续存储报修记录；

2. 管理员打开「报修统计分析」页面，页面自动加载统计接口；

3. 顶部展示报修总数、已处理 / 待处理数量、平均处理时长；

4. 柱状图自动展示电梯、电路、公共设施等报修分类数量；下方明细表格可筛选导出月度维修报表。

### 6\.5 物业收费催缴流程

1. 抄表生成水电、物业费记录，未缴费状态自动标记橙色；

2. 首页待办事项展示未缴费户数，点击「去查看」跳转物业收费页面；

3. 筛选未缴费记录，联系业主完成缴费后，点击「缴费」按钮更新状态；

4. 系统自动更新本月物业收入、物业费收缴率统计卡片。

## 七、系统部署操作手册

### 7\.1 环境前置准备

1. JDK 1\.8 及以上版本；

2. MySQL 8\.0，创建空数据库 `community_property_db`；

3. Maven 3\.6\+ 用于后端打包；

4. Node\.js 16\+ 用于前端依赖安装、打包。

### 7\.2 数据库初始化

1. 执行项目根目录 SQL 脚本，自动创建全部业务数据表、初始化测试数据（admin、mary 默认账号）；

2. 确认数据库字符集为 utf8mb4，引擎 InnoDB。

### 7\.3 后端配置修改

打开 `application.yml` 配置文件，修改以下配置：

1. MySQL 数据库连接地址、账号、密码；

2. 文件上传存储路径（如需上传附件扩展）；

3. 后端服务端口号（默认 8080）。

### 7\.4 后端打包与启动

1. 项目根目录执行 Maven 命令：`mvn clean package -DskipTests`；

2. 打包完成后 target 目录生成 jar 包；

3. 命令行运行：`java -jar xxx.jar` 启动后端接口服务。

### 7\.5 前端部署运行

1. 前端文件夹执行 `npm install` 安装全部依赖；

2. 本地开发调试：`npm run dev`；

3. 生产打包部署：`npm run build`，生成 dist 静态文件，部署至 Nginx。

### 7\.6 系统访问使用

浏览器输入前端访问地址，打开登录页面，选择管理员 / 业主角色，输入默认账号登录使用后台 / 前台功能。

## 九、项目总结与后期扩展优化方向

### 9\.1 项目总结

本智慧社区物业管理系统基于 SpringBoot\+Vue3 前后端分离架构开发，聚焦小区物业全场景业务需求，完整实现社区资产台账（楼宇 / 房屋 / 业主 / 车位）、水电抄表自动核算、物业 / 车位收费管理、业主投诉报修工单闭环、数据可视化统计、多角色权限管控全套功能。
项目技术选型成熟通用，MyBatis\-Plus 简化数据库开发，ECharts 实现业务数据可视化，全局拦截器保障账号权限安全，批量操作、多条件检索、首页待办预警等设计大幅降低物业手工办公成本，工单、收费、资产数据全程可追溯，解决传统线下物业纸质登记、核算繁琐、工单遗漏、数据无统计分析等痛点，可直接交付小区物业落地使用，也可作为课程设计、毕业设计完整项目交付。

### 9\.2 后期扩展优化方向

1. **业主移动端 H5 / 小程序**：开发手机端页面，业主手机提交投诉报修、在线缴纳物业费、车位费；

2. **消息通知功能**：新投诉、新报修、费用到期推送短信 / 站内消息提醒业主与管理员；

3. **文件附件上传**：投诉、报修支持上传现场图片，抄表支持上传水表照片；

4. **财务导出报表**：月度收费、报修统计支持 Excel 导出打印；

5. **门禁访客管理模块**：新增访客登记、小区门禁权限管理；

6. **维修人员绩效统计**：统计每位维修师傅月度处理工单数量，生成绩效报表；

7. **多小区多租户隔离**：支持系统同时管理多个不同小区，数据相互隔离；

8. **在线支付对接**：集成微信 / 支付宝支付接口，业主线上直接缴费，自动更新缴费状态。
